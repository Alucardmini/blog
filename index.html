自定义NSLog之pch文件作用

性能优化

Download as .zip Download as .tar.gz View on GitHub

iOS开发中，难免用到NSLog调试，真机调试中无处不在的NSLog已经功成但未身退，不仅增加开销还有在审核被拒绝的危险，因此建议自定义NSLog，只需在工程pch文件中添加如下：

ifdef DEBUG

define Log(...) NSLog(VA_ARGS)// VA_ARGS前后有两个下划线"__" git 的编译器为啥不识别呢？

else

define Log(...)

endif

P.s关于pch文件（以下引自李明杰老师） 1.项目的Supporting files文件夹下的“工程名-Prefix.pch”文件，是一个头文件

2.pch头文件的内容能被项目中的其他所有源文件共享和访问

3.一般在pch文件中定义一些全局的宏

4.在pch文件中添加下列预处理指令，然后在项目中使用Log(…)来输出日志信息，就可以在发布应用的时候，一次性将NSLog语句移除（在调试模式下，才有定义DEBUG）
